## 2026-03-29 - React Query N+1 Redundancy on Dashboards
**Learning:** I found a codebase-specific performance pattern in React Query dashboard widgets where multiple hooks (`useQuery`) fetch subsets of data from the exact same base table (e.g., fetching `employees` multiple times to count total headcount, find upcoming birthdays, and calculate attrition). Because `useQuery` calls are unbatched by default, this creates N+1-like network redundancies and unnecessary overhead on the frontend and the database.
**Action:** When working on dashboards or aggregate UI components using React Query, analyze whether multiple network calls to the same table can be combined into a single query fetching the necessary projection of columns. Fetch the base data once, and compute the derived widget statistics (e.g., headcount, department breakdown) locally using synchronous JavaScript (and optionally `useMemo`).
