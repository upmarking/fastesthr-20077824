## 2024-03-24 - HTML Injection in PDF Templates
**Vulnerability:** XSS/HTML Injection vulnerability in `src/lib/pdf-generator.ts` where unescaped user inputs (e.g., candidate names, custom template variables) were directly substituted into HTML strings before being rendered to PDF and persisted.
**Learning:** The PDF generator natively executes `<script>` tags found in templates for features like CTC calculations before capturing `.innerHTML`. Using robust sanitizers like DOMPurify on the full template would strip these required scripts and break the feature.
**Prevention:** Rather than sanitizing the entire compiled template block or pulling in heavy dependencies, escape individual variable values (e.g., `&`, `<`, `>`, `"`, `'`) specifically during variable substitution. This ensures user input is safe while preserving the template's intentional functional elements.

## 2024-05-18 - Missing HTML Sanitization in ID Card Templates
**Vulnerability:** ID Card HTML templates and viewer components (Virtual ID Card and Public ID Card) were injecting user-controlled HTML strings with `dangerouslySetInnerHTML` after performing simple string variable replacements (e.g. employee names, designation) without any HTML escaping. This leads to a High severity Stored XSS vulnerability where malicious code inside employee details could be executed when admins or other users view the generated cards.
**Learning:** Even when interpolating simple text variables into an HTML template string, the resulting strings must be escaped before rendering to prevent malicious HTML tags from being evaluated. Attempting to use a heavy sanitizer like DOMPurify may break inline CSS/styles and introduces unnecessary dependencies, violating project rules about adding new security dependencies when simpler solutions exist.
**Prevention:** Instead of pulling in large dependencies like DOMPurify to sanitize the entire rendered template block, simply escape the individual variable values (e.g., `&`, `<`, `>`, `"`, `'`) specifically during variable substitution. This ensures user input is safe while preserving the template's intentional functional elements and avoiding new dependency bloat.
