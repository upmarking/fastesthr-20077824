## 2024-05-24 - Unauthenticated Access to Edge Functions Bypassing RLS
**Vulnerability:** The `send-offer-letter` Supabase edge function accepted requests and performed sensitive operations (accessing/saving database records via service role key and sending external emails) without explicitly verifying the caller's identity or authorization. This exposed the platform to unauthenticated access, email spoofing, and data leakage risks.
**Learning:** Supabase edge functions that utilize `SUPABASE_SERVICE_ROLE_KEY` bypass all Row Level Security (RLS) policies by default. When an edge function acts on behalf of a user using this elevated privilege, it cannot rely on RLS for access control and must manually extract the `Authorization` header, resolve the token via `supabase.auth.getUser()`, and implement explicit authorization checks (such as verifying cross-tenant boundaries like `company_id`).
**Prevention:** Any edge function utilizing the service role key MUST include manual authentication (`supabase.auth.getUser()`) and verify tenant/resource boundaries before reading, writing, or executing external commands based on client inputs.
