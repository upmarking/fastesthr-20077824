## 2024-03-24 - HTML Injection in PDF Templates
**Vulnerability:** XSS/HTML Injection vulnerability in `src/lib/pdf-generator.ts` where unescaped user inputs (e.g., candidate names, custom template variables) were directly substituted into HTML strings before being rendered to PDF and persisted.
**Learning:** The PDF generator natively executes `<script>` tags found in templates for features like CTC calculations before capturing `.innerHTML`. Using robust sanitizers like DOMPurify on the full template would strip these required scripts and break the feature.
**Prevention:** Rather than sanitizing the entire compiled template block or pulling in heavy dependencies, escape individual variable values (e.g., `&`, `<`, `>`, `"`, `'`) specifically during variable substitution. This ensures user input is safe while preserving the template's intentional functional elements.
## 2024-05-24 - Cross-Site Scripting (XSS) & Regex Substitution Vulnerabilities in Dynamic Templates
**Vulnerability:** Document template renderers (`VirtualIDCard`, `PublicIDCard`, `OfferLetterRenderer`, `DocumentRenderer`) were directly replacing placeholders with raw values and injecting them into `dangerouslySetInnerHTML`. Furthermore, the string `.replace` method used string substitution which parses `$&` and `$n` natively, allowing malicious user input to manipulate structural elements of the template.
**Learning:** React components dealing with customizable HTML templates, like offer letters or ID cards, are heavily susceptible to XSS if not properly sanitized. Also, passing untrusted strings directly into the second parameter of `String.prototype.replace(regex, string)` is a subtle but potent injection vector.
**Prevention:** Always wrap dynamically generated HTML with `DOMPurify.sanitize(html)` before rendering. In DOMPurify configs involving visual templates, preserve styles via `ADD_TAGS: ['style']` and `ADD_ATTR: ['style']`. Additionally, use the callback pattern for `String.prototype.replace(regex, () => value)` when replacing strings with unknown/user-controlled data to disable native regex backreference substitution.
