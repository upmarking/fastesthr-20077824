## 2024-03-24 - HTML Injection in PDF Templates
**Vulnerability:** XSS/HTML Injection vulnerability in `src/lib/pdf-generator.ts` where unescaped user inputs (e.g., candidate names, custom template variables) were directly substituted into HTML strings before being rendered to PDF and persisted.
**Learning:** The PDF generator natively executes `<script>` tags found in templates for features like CTC calculations before capturing `.innerHTML`. Using robust sanitizers like DOMPurify on the full template would strip these required scripts and break the feature.
**Prevention:** Rather than sanitizing the entire compiled template block or pulling in heavy dependencies, escape individual variable values (e.g., `&`, `<`, `>`, `"`, `'`) specifically during variable substitution. This ensures user input is safe while preserving the template's intentional functional elements.
## 2025-04-18 - Prevent XSS and Regex Injection when rendering user inputs into React `dangerouslySetInnerHTML`
**Vulnerability:** HIGH severity XSS and Regex Injection vulnerability existed because user-controlled data was directly substituted into HTML template strings via `String.prototype.replace()` without escaping HTML characters. Additionally, `html.replace()` was directly interpolating strings instead of using a callback, meaning user input like `$&` could trigger unintended Regex backreference substitutions.
**Learning:** `dangerouslySetInnerHTML` allows for XSS if unescaped variables are interpolated into the source. Even simple variable substitution poses a threat of regex injection if `String.prototype.replace()` is called directly with a string rather than a functional callback. Lightweight HTML escaping logic without adding new dependencies is effective when replacing simple strings.
**Prevention:** Always HTML-escape variables substituting into HTML templates rendered via `dangerouslySetInnerHTML`. Use functional callbacks `() => escapedValue` instead of passing strings directly to `replace()` to prevent unintended Regex backreference substitutions.
